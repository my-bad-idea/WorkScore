# 测试岗位考核标准（基于软件部周报汇总）- 评分一致优化版

适用对象：功能测试 / 业务测试 / 回归测试 / 测试用例维护与迁移 / 兼顾部分工具与自动化建设的测试岗位。

---

## 【评分输出约定】必读

- 评分时必须且仅能输出以下 **5 个考核项**，不得增删、不得改名，顺序可任意：
  1. **用例资产与覆盖**
  2. **缺陷有效性与质量**
  3. **执行回归与上线**
  4. **协作与支撑**
  5. **工作类型适配**
- 每项 0–100 分，按下方「各考核项评分区间」严格对照后给分；**必须**在 comment 中写清扣分理由或亮点（一句话即可）。
- 若周报中**未提及**某维度（如无任何缺陷/用例相关描述），该项按「无描述」档给分，并在 comment 中注明「周报未涉及」。
- **不唯缺陷数量**：提交 Bug 多不等于贡献大，更关注有效缺陷率、严重性、发现时机、漏测率；无缺陷数据时该项不得凭主观打高分。

## 各考核项评分区间（严格按此对照）

### 1. 用例资产与覆盖

| 分数区间 | 等级 | 判定条件（满足即落入该档） |
|----------|------|----------------------------|
| 85–100   | 优秀 | 有用例新增/更新/迁移数量或覆盖率数据；有需求-用例追溯、评审结论或抽检结果；可执行、关键路径覆盖可验证。 |
| 70–84    | 良好 | 有具体用例数量或覆盖范围描述；有模块/需求对应关系或评审记录之一；略缺量化对比。 |
| 55–69    | 一般 | 仅泛泛提到“写用例”“维护用例”“迁移”等，无数量、无覆盖范围、无评审/抽检结论。 |
| 0–54     | 不足 | 几乎无用例相关描述；或仅“参与测试”“执行测试”等无法区分用例产出的表述。 |

- **无描述**：周报完全未提用例/覆盖率/迁移/评审 → 给 0–30 分，comment 写「周报未涉及用例资产与覆盖」。

### 2. 缺陷有效性与质量

| 分数区间 | 等级 | 判定条件 |
|----------|------|----------|
| 85–100   | 优秀 | 有有效缺陷率、缺陷等级分布或缺陷单号+复现步骤/日志；有重复/无效占比或闭环周期数据；缺陷描述可定位、可复现。 |
| 70–84    | 良好 | 有缺陷数量与等级（如 P0/P1/P2）或部分单号；有有效/无效/回归结论之一；信息较完整。 |
| 55–69    | 一般 | 仅提到“提了若干 bug”“修复缺陷”等，无等级、无有效性、无复现信息。 |
| 0–54     | 不足 | 无缺陷相关描述；或仅主观表述“发现很多问题”等无数据无单号。 |

- **无描述**：未提缺陷/有效率/等级/单号 → 给 0–30 分，comment 写「周报未涉及缺陷有效性与质量」。

### 3. 执行回归与上线

| 分数区间 | 等级 | 判定条件 |
|----------|------|----------|
| 85–100   | 优秀 | 有执行用例数/计划数、版本号/环境、回归清单、准入结论或上线验证结果；关键路径/高风险覆盖可追溯。 |
| 70–84    | 良好 | 有测试范围、版本或环境、执行结果或回归/准入结论其中至少两项；有简单数据。 |
| 55–69    | 一般 | 仅“执行测试”“做了回归”“参与上线”等笼统描述，无范围、无版本、无清单。 |
| 0–54     | 不足 | 无执行/回归/上线相关描述；或无法判断是否完成闭环。 |

- **漏测**：若周报提到上线后 P0/P1 漏测且未说明根因与改进 → 该项不超过 75 分，comment 中须提及漏测与改进点。
- **无描述**：未提执行/回归/准入/上线验证 → 给 0–30 分，comment 写「周报未涉及执行回归与上线」。

### 4. 协作与支撑

| 分数区间 | 等级 | 判定条件 |
|----------|------|----------|
| 85–100   | 优秀 | 有阻塞问题同步/联调/回归反馈的时效或闭环记录；有环境、数据准备或现场协助的周期/结果说明。 |
| 70–84    | 良好 | 提到阻塞处理、联调配合、环境数据或现场支撑其中至少 1 项，有简要结果或时效。 |
| 55–69    | 一般 | 仅“配合联调”“协助环境”等笼统表述，无记录、无时效。 |
| 0–54     | 不足 | 无协作或支撑相关描述。 |

- **无描述**：未提协作/阻塞/环境/数据/现场 → 给 0–30 分，comment 写「周报未涉及协作与支撑」。

### 5. 工作类型适配

| 分数区间 | 等级 | 判定条件 |
|----------|------|----------|
| 85–100   | 优秀 | 能识别主工作类型（用例设计/执行回归/缺陷分析/自动化建设/环境支撑）且描述与证据匹配；有阶段或占比说明。 |
| 70–84    | 良好 | 工作类型清晰，内容与类型基本匹配，略缺占比或阶段说明。 |
| 55–69    | 一般 | 有工作内容但类型不清晰或与证据不匹配。 |
| 0–54     | 不足 | 无法判断工作类型；或内容过空无法适配。 |

- **无描述**：周报过短或过空无法判断 → 给 0–30 分，comment 写「周报内容过少无法判断类型」。

## 证据与数据来源（供对照周报内容）

评分时从周报中识别以下“证据字段”（满足越多、描述越具体，对应维度分数越高）：

- **用例**：新增/更新数量、覆盖模块、评审结论、迁移完成度、需求-用例追溯
- **执行**：测试范围、版本号/环境、执行结果、阻塞项、关键路径覆盖清单
- **缺陷**：缺陷单号、等级、复现步骤/截图/日志、有效/无效/关闭与回归结论
- **回归/上线**：回归清单、准入结论、上线验证结果、漏测（P0/P1）与归因
- **数据/环境**：测试数据构造、环境变更记录、现场协助与闭环

**统一原则**：以“用例、缺陷、回归、报告、数据与环境记录”为证据；无数据无单号时对应维度不得打高分（优秀档需有至少 2 类证据或 3 项以上可追溯项）。

---

## 阶段/工作类型与权重参考（供综合判断）

- **A 用例设计/维护/迁移**：用例覆盖率、评审通过率、可执行率、迁移进度
- **B 功能测试/回归/验收**：执行完成率、关键路径覆盖、阻断缺陷发现时机、上线验证质量
- **C 缺陷管理与质量分析**：缺陷闭环效率、重复/无效率、根因归类与复测质量
- **D 自动化/工具/效率**：自动化覆盖、脚本稳定性、工具沉淀
- **E 环境与数据/现场支撑**：数据准备效率、环境问题处理、客户/现场协助

不同时间段可侧重不同类型（评审期重 A、执行期重 B、上线期重 B+C、建设期重 D）；「工作类型适配」项体现当期主类型及占比；若周报未区分类型，则按描述整体判断后给分。

---

## 结果输出建议（供被考核人改进周报）

- 本周期覆盖模块、版本与环境（明确边界）
- 用例资产：新增/更新/迁移（关键数字 + 抽检结论）
- 缺陷质量：有效缺陷率、阻断缺陷、漏测（关键数字 + 典型缺陷单号）
- 上线与回归：回归清单与准入结论（可追溯）
- 质量改进：Top 1 复盘/建议及其验证口径
